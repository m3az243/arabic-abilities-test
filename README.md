# اختبار القدرات التجريبي

منصة ويب أمامية فقط تحاكي اختبار القدرات العامة بأسلوب سريع ومريح، مع واجهة عربية `RTL`، مؤقت دقيق، حفظ تلقائي، نتائج فورية، وتحليل أداء مفصل بدون أي مكتبات خارجية.

## نظرة عامة

المشروع مخصص لعرض اختبار تفاعلي تجريبي في القسمين:
- اللفظي
- الكمي

ويعمل بالكامل في المتصفح باستخدام:
- `HTML`
- `CSS`
- `Vanilla JavaScript`
- `LocalStorage`
- `JSON`

## الصفحات

- `index.html`
  صفحة البداية، وفيها بدء الاختبار أو استئنافه وتبديل الثيم.

- `exam.html`
  صفحة تنفيذ الاختبار، وفيها المؤقت، شريط التقدم، اختيار الإجابات، التعليم للمراجعة، والتنقل بين الأسئلة.

- `results.html`
  صفحة النتائج النهائية، وفيها الدرجة، النسبة، التقييم، التحليل حسب القسم والنوع، ومراجعة الأسئلة.

- `about.html`
  صفحة تعريفية بالفريق المنفذ للمشروع.

## المزايا الرئيسية

- تصميم عربي `RTL` مناسب للجوال والتابلت والكمبيوتر
- 4 ثيمات:
  الأزرق الهادئ، الرملي الدافئ، القراءة المريحة، الداكن
- حفظ تلقائي أثناء الاختبار عبر `localStorage`
- استئناف المحاولة بعد إعادة فتح الصفحة
- مؤقت دقيق لمدة 25 دقيقة
- تعليم الأسئلة للمراجعة
- تتبع الوقت المستغرق لكل سؤال
- نتائج فورية مع تحليل حسب القسم والنوع الفرعي
- مراجعة تفصيلية بعد الانتهاء
- طباعة النتائج مباشرة من المتصفح
- عمل محلي بدون خادم خلفي
- دعم fallback للأسئلة عند فتح الملفات محليًا

## تشغيل المشروع محليًا

### الطريقة السريعة

افتح الملف:

```text
index.html
```

مباشرة في المتصفح.

### ملاحظة

المشروع يحتوي على fallback داخلي للأسئلة، لذلك سيعمل حتى عند فتح الملفات محليًا بدون سيرفر.

## النشر على GitHub Pages

بعد رفع المشروع إلى GitHub:

1. افتح المستودع.
2. ادخل إلى `Settings`.
3. افتح `Pages`.
4. اختر:
   - `Deploy from a branch`
   - الفرع `main`
   - المجلد `/ (root)`
5. اضغط `Save`.

بعد ذلك سيظهر رابط الموقع المنشور.

## تحديث الموقع بعد أي تعديل

من داخل مجلد المشروع في الطرفية:

```powershell
git add .
git commit -m "Update project"
git push
```

ثم انتظر قليلًا حتى يتحدث الموقع المنشور.

## التخزين المحلي

يستخدم المشروع المفاتيح التالية داخل `localStorage`:

- `examState`
  لحفظ حالة الاختبار الحالية

- `examHistory`
  لحفظ سجل المحاولات

- `settings`
  لحفظ تفضيلات المستخدم مثل الثيم

## فحص المشروع قبل التسليم

- تأكد من عمل الصفحات الأربع بدون أخطاء Console
- اختبر التجربة على الجوال والكمبيوتر
- جرّب بدء الاختبار ثم تحديث الصفحة للتأكد من الاستئناف
- جرّب إنهاء الاختبار ومراجعة النتائج
- جرّب تغيير الثيمات والتأكد من حفظها
